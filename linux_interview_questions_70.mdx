# Linux Interview Questions and Answers (150)

## What is Linux?

Linux is an open-source Unix-like operating system kernel used in servers, cloud platforms, containers, and embedded systems.

## What is a Linux distribution?

A Linux distribution combines the Linux kernel with packages, libraries, tools, and package managers.

## What is the Linux kernel?

The kernel is the core component that manages hardware, memory, processes, and devices.

## What is the root user?

The root user has unrestricted administrative privileges on the system.

## What is a shell?

A shell is a command-line interpreter that allows users to interact with the operating system.

## What is Bash?

Bash is the most widely used Linux shell and supports scripting and automation.

## What is an inode?

An inode stores file metadata such as ownership, permissions, timestamps, and disk block locations.

## What is the difference between Linux and Unix?

Unix is generally proprietary while Linux is open source and community-driven.

## How do you check the Linux version?

Use uname -a, hostnamectl, or cat /etc/os-release.

## What is a filesystem?

A filesystem organizes how data is stored and retrieved from storage devices.

## What is swap space?

Swap is disk space used as virtual memory when RAM becomes insufficient.

## What is a process?

A process is a running instance of a program.

## What is a daemon?

A daemon is a background service that runs without user interaction.

## What is the purpose of /etc?

It stores system-wide configuration files.

## What is the purpose of /var?

It stores logs, spool files, caches, and other variable data.

## What is the purpose of /tmp?

It stores temporary files created by users and applications.

## What is load average?

It represents the average number of processes waiting for CPU resources.

## What is the PATH variable?

PATH defines directories searched for executable commands.

## What is open source software?

Software whose source code is freely available for use and modification.

## How do you get help for a command?

Use man command_name or command_name --help.

## What does pwd do?

Displays the current working directory.

## What does ls do?

Lists files and directories.

## How do you list hidden files?

Use ls -la.

## What does cd do?

Changes the current directory.

## How do you create a file?

Use touch filename.

## How do you create a directory?

Use mkdir directory_name.

## How do you remove a file?

Use rm filename.

## How do you remove a directory?

Use rm -r directory_name.

## What does cp do?

Copies files and directories.

## What does mv do?

Moves or renames files and directories.

## What does cat do?

Displays file contents.

## What does tac do?

Displays file contents in reverse order.

## What does head do?

Shows the first lines of a file.

## What does tail do?

Shows the last lines of a file.

## How do you monitor a growing log file?

Use tail -f logfile.

## What does grep do?

Searches text for matching patterns.

## How do you ignore case in grep?

Use grep -i.

## What does find do?

Searches files and directories based on criteria.

## What does locate do?

Searches files using a prebuilt database.

## What does awk do?

Processes and extracts structured text data.

## What does sed do?

Performs text transformations and substitutions.

## What does sort do?

Sorts lines of text.

## What does uniq do?

Removes adjacent duplicate lines.

## What does cut do?

Extracts selected fields from text.

## What does tr do?

Translates or deletes characters.

## What does wc do?

Counts lines, words, and characters.

## What does xargs do?

Builds and executes commands from standard input.

## What does tee do?

Writes output to both screen and file.

## What does tar do?

Archives multiple files into a single file.

## How do you compress files with gzip?

Use gzip filename.

## What does zip do?

Creates compressed zip archives.

## What does df do?

Displays filesystem disk usage.

## What does du do?

Displays directory and file sizes.

## What does uname do?

Displays system information.

## What does history do?

Shows previously executed commands.

## What does which do?

Shows the location of a command executable.

## What does whereis do?

Locates binaries, source files, and man pages.

## What does alias do?

Creates command shortcuts.

## What does curl do?

Transfers data from or to servers.

## What does wget do?

Downloads files from the web.

## What does chmod do?

Changes file permissions.

## What does chown do?

Changes file ownership.

## What does chgrp do?

Changes file group ownership.

## What are Linux permissions?

Read, write, and execute permissions for user, group, and others.

## What does chmod 777 mean?

Gives full permissions to everyone and should be used cautiously.

## What does chmod 755 mean?

Owner gets full access; group and others get read and execute.

## What is umask?

Defines default permissions for new files and directories.

## How do you create a user?

Use useradd username.

## How do you set a password?

Use passwd username.

## How do you delete a user?

Use userdel username.

## How do you modify a user?

Use usermod.

## How do you create a group?

Use groupadd groupname.

## How do you add a user to a group?

Use usermod -aG group user.

## What is sudo?

Allows permitted users to execute commands as another user.

## What is the sudoers file?

It defines sudo permissions for users and groups.

## How do you view running processes?

Use ps, top, or htop.

## What is a PID?

A PID is a unique process identifier.

## How do you kill a process?

Use kill PID.

## What does kill -9 do?

Forcefully terminates a process using SIGKILL.

## What is top?

A real-time process and resource monitoring tool.

## What is htop?

An interactive and user-friendly process viewer.

## What is a zombie process?

A completed process waiting for its parent to read the exit status.

## What is an orphan process?

A process whose parent has terminated.

## What does nice do?

Starts a process with a specified priority.

## What does renice do?

Changes the priority of a running process.

## What is systemd?

The service manager used by most modern Linux distributions.

## How do you start a service?

Use systemctl start service.

## How do you stop a service?

Use systemctl stop service.

## How do you restart a service?

Use systemctl restart service.

## How do you check service status?

Use systemctl status service.

## How do you enable a service at boot?

Use systemctl enable service.

## How do you disable a service?

Use systemctl disable service.

## What is journalctl?

A tool used to view systemd logs.

## How do you view boot logs?

Use journalctl -b.

## How do you view logs for a service?

Use journalctl -u service_name.

## What is an IP address?

A unique identifier assigned to a network device.

## How do you view network interfaces?

Use ip addr show.

## What does ping do?

Tests network connectivity.

## What does traceroute do?

Shows the path packets take to a destination.

## What is DNS?

A service that translates domain names into IP addresses.

## How do you query DNS records?

Use dig or nslookup.

## What does netstat do?

Displays network connections and listening ports.

## What command replaces netstat?

The ss command.

## How do you check open ports?

Use ss -tulnp.

## What is SSH?

A secure protocol for remote access.

## How do you generate SSH keys?

Use ssh-keygen.

## What is SCP?

A secure file copy utility over SSH.

## What is rsync?

A tool for efficient file synchronization.

## What is a firewall?

Software that controls network traffic.

## What is firewalld?

A dynamic firewall management service.

## What is LVM?

Logical Volume Manager provides flexible disk management.

## What is a physical volume?

The underlying storage device used by LVM.

## What is a volume group?

A storage pool created from physical volumes.

## What is a logical volume?

A virtual partition created from a volume group.

## How do you check mounted filesystems?

Use mount or df -h.

## What is /etc/fstab?

Defines filesystems mounted at boot.

## What is RAID?

A technology combining disks for redundancy or performance.

## What is RAID 0?

Disk striping without redundancy.

## What is RAID 1?

Disk mirroring with redundancy.

## How do you check disk usage?

Use df -h and du -sh.

## What is a shell script?

A file containing Linux commands executed sequentially.

## How do you make a script executable?

Use chmod +x script.sh.

## What is a shebang?

The first line specifying the interpreter, such as #!/bin/bash.

## What is a variable in shell scripting?

A named value used within a script.

## How do you read user input?

Use the read command.

## What is an if statement?

A conditional statement that executes code based on conditions.

## What is a for loop?

A loop that iterates over a sequence of values.

## What is a while loop?

A loop that runs while a condition remains true.

## What is $? in shell scripting?

It stores the exit status of the previous command.

## What are positional parameters?

Arguments passed to a script such as $1, $2, and $3.

## Server disk is full. What do you do?

Use df, du, and log analysis to locate large files and free space.

## High CPU usage troubleshooting steps?

Identify heavy processes using top and analyze application behavior.

## High memory usage troubleshooting steps?

Use free, vmstat, and top to find memory-consuming processes.

## A service is down. What do you check first?

Check systemctl status and relevant logs.

## SSH is not working. What do you check?

Verify network connectivity, firewall rules, SSH service status, and logs.

## A website is slow. What do you investigate?

Check CPU, memory, disk, network, and application logs.

## Users cannot resolve domain names. What do you check?

Verify DNS configuration and name server reachability.

## A filesystem is read-only. What do you do?

Check disk health, mount status, and kernel logs.

## Logs are growing rapidly. What should you do?

Investigate the root cause and configure log rotation.

## How do you troubleshoot boot failures?

Review boot logs, filesystem integrity, and service dependencies.

## Why is Linux important for DevOps?

Most cloud, container, and CI/CD environments run on Linux.

## How is Linux used with Docker?

Docker containers share the Linux kernel and rely on Linux namespaces and cgroups.

## How is Linux used in Kubernetes?

Worker nodes and containers commonly run on Linux systems.

## Why is shell scripting important in DevOps?

It automates repetitive operational tasks.

## How do you schedule jobs in Linux?

Use cron for recurring tasks.

## What is CI/CD?

Continuous Integration and Continuous Delivery automate software build and deployment.

## How do you monitor Linux servers?

Use tools such as Prometheus, Grafana, top, and system logs.

## Why are logs important in DevOps?

They help diagnose issues and track application behavior.

## How do you secure Linux servers?

Apply least privilege, patch regularly, use firewalls, and secure SSH access.

## What Linux skills should every DevOps engineer know?

Commands, networking, scripting, troubleshooting, services, storage, security, containers, and cloud fundamentals.
